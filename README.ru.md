Бот на TypeScript для торговли на рынках Polymarket по **дневному максимуму
температуры** («Будет ли максимальная температура в Сеуле 29°C 19 июня?»),
использующий **вероятностный прогноз** вместо точечной оценки.

Он делает то, что опытные «погодные» трейдеры делают вручную: строит
*распределение* дневного максимума температуры, превращает его в вероятность для
каждого °C-интервала рынка, сравнивает с ценой в реальном стакане и делает ставку —
выборочно и с расчётом размера — только там, где модель и рынок существенно
расходятся.

```
ансамблевый прогноз ──► распределение по °C-интервалам ──► P(интервал)
                                                              │
стакан в реальном времени ──► рыночная цена интервала ────────┤
                                                              ▼
                                  преимущество = P(интервал) − цена
                                  │  (фильтр по мин. преимуществу / цене / ликвидности)
                                  ▼
                  ставка по дробному Келли ──► лимиты риска ──► ордер
```

## Как это работает

| Этап | Модуль | Что делает |
|------|--------|------------|
| Поиск | [src/polymarket/gamma.ts](src/polymarket/gamma.ts) | Находит активные события «максимальная температура в {городе}» через Gamma API `public-search` и разбирает каждый рынок Yes/No в температурный **интервал** (`≤24`, `=25`, … `≥34`). |
| Прогноз | [src/weather/forecast.ts](src/weather/forecast.ts) | Берёт **ансамбль Open-Meteo** (≈140 членов по 4 моделям), вычисляет дневной максимум каждого члена и строит распределение через гауссово ядро. Для городов США рецентрируется на детерминированный максимум **NWS**. |
| Интервалы | [src/strategy/buckets.ts](src/strategy/buckets.ts) | Сопоставляет каждый интервал реальному температурному отрезку `[lo, hi)`, который его «закрывает», с учётом правила округления города. |
| Преимущество | [src/strategy/edge.ts](src/strategy/edge.ts) | Сравнивает модельное P(интервал) с ценой ask в реальном времени (обе стороны, Yes и No) и оставляет кандидатов, прошедших фильтры преимущества/цены/ликвидности. |
| Размер | [src/strategy/sizing.ts](src/strategy/sizing.ts) | Ставка по дробному Келли: `f* = (q − p)/(1 − p)`, умноженная на `KELLY_FRACTION`. |
| Риск | [src/risk/guards.ts](src/risk/guards.ts) | Лимиты на рынок, на событие, на суммарную экспозицию и на дневной убыток. |
| Исполнение | [src/trading/executor.ts](src/trading/executor.ts) | Симулирует исполнение («бумажный» режим) или выставляет реальные ордера на CLOB в боевом режиме. |
| Учёт | [src/trading/portfolio.ts](src/trading/portfolio.ts) + [src/store/db.ts](src/store/db.ts) | Позиции, экспозиция и реализованный PnL сохраняются в `data/state.json`. |

## Установка

```bash
npm install
cp .env.example .env        # затем отредактируйте .env
```

Отредактируйте `.env`. У всего есть безопасные значения по умолчанию, а
`TRADING_MODE=paper`, поэтому команды только для чтения можно запускать вообще
без кошелька.

## Команды

```bash
npm run forecast -- hong_kong          # вывести распределение температуры
npm run forecast -- nyc 2026-06-20     # на конкретную дату

npm run scan                           # анализ всех рынков, показ преимуществ и
                                       # намеченных сделок — ордера не выставляются

npm run run-once                       # один проход с исполнением (бумажный или боевой)
npm run watch -- 10                    # повторять проход каждые 10 минут

npm run auth                           # вывести CLOB API-ключи для кэширования в .env
npm run portfolio                      # позиции, экспозиция, реализованный PnL
```

`scan` всегда только для чтения и не требует ключей — начните с него.

## Переход к боевой торговле

1. Пополните **отдельный** кошелёк Polygon в USDC (не основной кошелёк).
2. Задайте в `.env` `PRIVATE_KEY`, `FUNDER_ADDRESS` (адрес вашего профиля
   Polymarket) и `SIGNATURE_TYPE` (`0` EOA / `1` Magic-email / `2` Safe).
3. Один раз запустите `npm run auth` и вставьте выведенные ключи обратно в `.env`.
4. Установите `BANKROLL` не больше вашего реального баланса и держите
   `KELLY_FRACTION` низким (0.25 = четверть-Келли).
5. Переключите `TRADING_MODE=live`. Без ключей бот откажется запускаться в боевом
   режиме.

## ⚠️ Откалибруйте, прежде чем доверять

Качество прогноза не выше, чем его согласованность с **официальной станцией
расчёта**. Две самые важные «ручки» в [src/weather/cities.ts](src/weather/cities.ts):

- **`rounding`** — станция сообщает *округлённое* (`26` = 25.5–26.5) или
  *усечённое* целое? Неверный выбор смещает каждую вероятность.
- **`biasC`** — небольшая аддитивная поправка, если температура модели на высоте
  2 м систематически выше/ниже микроклимата станции.

При запуске `scan` вы иногда увидите **очень большие** расхождения модели и рынка.
Воспринимайте их как *повод проверить калибровку*, а не как бесплатные деньги —
рынок часто знает особенности станции лучше, чем неоткалиброванная модель.
Используйте `archiveDailyHighs()` в
[src/weather/openmeteo.ts](src/weather/openmeteo.ts), чтобы протестировать вывод
модели на исторических официальных максимумах и настроить `biasC` до того, как
рискнёте капиталом.

## Встроенные защиты

Включены по умолчанию и настраиваются в `.env`:

- **Предел разумности преимущества** (`EDGE_SANITY_CAP`, по умолчанию 0.35) —
  преимущества больше этого считаются *ошибкой модели* и пропускаются, чтобы бот
  никогда не ставил больше всего на самый «неправильно оценённый» сигнал.
- **Раздувание разброса** (`SPREAD_INFLATION`, по умолчанию 1.4) — ансамбли
  дневного максимума недодисперсны; это расширяет прогнозное распределение, чтобы
  модель перестала создавать ложную уверенность.
- **Внутридневной пол** — для рынков «на сегодня» распределение ограничивается
  снизу уже наблюдавшейся сегодня температурой (максимум не может оказаться ниже).
- **Защита у расчёта** (`MIN_MINUTES_TO_RESOLVE`, по умолчанию 60) — никогда не
  торгует событие, которое вот-вот рассчитается.
- **Лимиты без утечек** — лимиты на рынок, событие и суммарную экспозицию
  учитывают как уже зафиксированные позиции, *так и* ставки, выделенные ранее в
  том же проходе, поэтому повторные проходы `watch` не могут превысить потолки.

Это ПО предназначено для исследовательских/образовательных целей. Торговля несёт
реальный финансовый риск; вы сами отвечаете за свои средства и решения.

## Примечания

- Зависимость `@polymarket/clob-client` заархивирована, но остаётся самым
  проверенным способом торговать на боевом CLOB сегодня; она изолирована за
  [src/polymarket/clob.ts](src/polymarket/clob.ts), поэтому будущий переход на
  новый унифицированный SDK Polymarket затронет только этот файл.
- Платные сервисы не нужны: Open-Meteo и NWS бесплатны и не требуют ключей.
